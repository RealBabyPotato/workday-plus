# tests

this folder is for project-level tests.

what goes here:

- end-to-end tests
- future cross-app integration tests

why this exists:

- some bugs only show up when frontend and backend work together
- this folder keeps those tests separate from app-specific tests

install:

- no separate install
- install the relevant app tools first, then run tests from here when they are added

launch:

- run project-level tests from this folder once they are added
