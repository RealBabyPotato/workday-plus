# e2e

this folder is for end-to-end tests, most likely with playwright.

what goes here:

- user flows like log in, plan courses, build a schedule, and save it

why this exists:

- unit tests check pieces
- e2e tests check the real student workflow

install:

- no separate install
- install frontend and backend tooling first, then add and run playwright from this folder later

launch:

- run end-to-end tests from here after both frontend and backend are running
