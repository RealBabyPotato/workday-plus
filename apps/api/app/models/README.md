# models

this folder is for backend data models.

what goes here:

- user
- course
- section
- meeting time
- degree rule
- saved plan

hello-world file:

- `course.py` is the smallest example of a backend data model

why this exists:

- shared data definitions are the backbone of the backend
- many backend teammates can work here without touching frontend code

install:

- no separate install
- install the backend app from [apps/api](../../README.md)

launch:

- no separate launch
- model files are imported by backend features and database code
