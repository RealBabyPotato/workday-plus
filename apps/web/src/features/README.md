# features

this folder is for feature-based frontend code.

example features:

- planner
- courses
- schedules
- degree-progress

hello-world file:

- `hello-world/HelloWorldCard.tsx` is the smallest feature folder example

why this exists:

- it lets the team organize code around product features instead of random file types
- that usually scales better once multiple people work on the frontend

install:

- no separate install
- install the frontend app from [apps/web](../../README.md)

launch:

- no separate launch
- feature code runs inside pages and components when the frontend is open
