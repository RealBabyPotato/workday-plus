# pages

this folder is for screen-level ui.

what goes here:

- dashboard page
- planner page
- schedule page
- degree progress page

hello-world file:

- `HomePage.tsx` is the basic screen example

why this exists:

- pages are easier to reason about when they are separate from reusable components
- it gives frontend teammates obvious ownership over screens

install:

- no separate install
- install the frontend app from [apps/web](../../README.md)

launch:

- no separate launch
- page files render when the frontend app is running
