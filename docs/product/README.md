# product

this folder is for feature planning.

what goes here:

- mvp scope
- user stories
- priority lists
- feature cuts

example files:

- [mvp-scope.md](mvp-scope.md): a sample way to describe must-have vs later features
- [user-stories-example.md](user-stories-example.md): a sample way to write short product stories

why this exists:

- it keeps the team honest about scope
- it helps stop "cool ideas" from quietly becoming required work

install:

- no separate install
- this folder is for reading and writing project plans

launch:

- no launch
- this folder is documentation only

good format for product docs:

- what problem are we solving?
- who is this for?
- what is in v1?
- what is out of scope?
- what can wait until later?
