# mvp scope

this file shows one reasonable first version of the product.

the point of an mvp is not to do everything. the point is to ship the smallest version that is actually useful.

## target user

ubc students who want help planning courses and understanding degree progress.

## core problem

students often do not know:

- what courses they still need
- what they are allowed to take next
- how sections fit into their schedule
- how to plan future terms without manually checking many pages

## must-have features for v1

these are the features i would strongly recommend for the first real version:

1. student can choose a program or major
2. student can enter completed and current courses
3. app can show remaining requirements at a basic level
4. app can show possible next courses based on prerequisites
5. student can build a term schedule from sections
6. app can show a simple weekly schedule view
7. student can save a plan

## good stretch features

these are useful, but they should not block the first version:

- auto-generated schedules from preferences
- building map links
- elective suggestions
- professor notes or external professor links
- term-by-term degree planning view

## features to avoid in early scope

these are likely to add a lot of complexity too early:

- deep rate my professor integration
- full workday integration
- notifications
- advanced recommendation systems
- c++ scheduling engine as a hard requirement

## example feature cuts

if the team falls behind, cut in this order:

1. cut map integration
2. cut professor integration
3. cut auto-scheduler
4. cut advanced recommendation logic

do not cut the basic planning flow first. that is the core of the product.

## success criteria for v1

v1 is successful if a student can:

1. enter what they have already taken
2. see what they still need
3. see some valid next-course options
4. build a basic term schedule
5. save that plan

## note for the team

a lot of student projects fail because the idea is too broad, not because the team is weak.

scope control is part of engineering quality.
