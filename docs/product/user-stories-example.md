# user stories example

this file shows a simple way to write user stories.

keep them short.

## examples (i luv 210)

- as a student, i want to enter completed courses so i can see what requirements are left
- as a student, i want to view available sections so i can build a workable schedule
- as a student, i want to save a draft plan so i can come back to it later
- as a student, i want to avoid invalid course choices so i do not waste registration time

## why this format helps

- it keeps the feature tied to a user
- it explains the value of the feature
- it helps both design and backend understand the goal

## simple template

- as a `<type of user>`, i want to `<do something>`, so that `<i get some value>`
