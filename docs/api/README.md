# api

this folder is for backend api documentation.

what goes here:

- endpoint lists
- example requests
- example responses
- auth notes later

example files:

- [courses-list-example.md](courses-list-example.md): a sample endpoint doc the team can copy
- [api-entry-template.md](api-entry-template.md): a small template for future endpoint docs

why this exists:

- frontend and backend work moves faster when the api is written down
- it gives both sides a shared contract before code is finished

install:

- no separate install
- this folder is for reading and writing api docs

launch:

- no launch
- this folder is documentation only

good format for api docs:

- route and method
- purpose
- request shape
- response shape
- error cases
- notes for frontend and backend
