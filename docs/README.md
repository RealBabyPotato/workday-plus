# docs

this folder is for project knowledge.

why this exists:

- student teams lose time when decisions only live in discord messages
- docs make product and architecture decisions easier to share and revisit

folders here:

- `product`: scope, features, priorities
- `architecture`: technical decisions and tradeoffs
- `api`: endpoint notes and data contracts

good starting files:

- [product/mvp-scope.md](product/mvp-scope.md): example feature scope and cuts
- [architecture/system-overview.md](architecture/system-overview.md): example high-level system explanation
- [architecture/0001-single-fastapi-app.md](architecture/0001-single-fastapi-app.md): example architecture decision record
- [api/courses-list-example.md](api/courses-list-example.md): example endpoint documentation

install:

- no separate install
- everyone should read this folder, but nothing here needs to be run

launch:

- no launch
- docs are for reading and writing project knowledge
