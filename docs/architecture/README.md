# architecture

this folder is for technical decisions.

what goes here:

- why we chose fastapi
- why we kept one backend app
- database design notes
- scheduling engine design later

example files:

- [system-overview.md](system-overview.md): a sample high-level architecture summary
- [0001-single-fastapi-app.md](0001-single-fastapi-app.md): a sample architecture decision record

why this exists:

- architecture should be explained, not just implied by code
- this helps new teammates ramp up faster

install:

- no separate install
- this folder is for reading and writing technical notes

launch:

- no launch
- this folder is documentation only

good format for architecture docs:

- context: what problem are we solving?
- decision: what did we choose?
- why: why is this the best fit right now?
- tradeoffs: what do we lose?
- follow-up: what should we revisit later?
